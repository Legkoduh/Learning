Веб-сервис - это программа, которая организовывает взаимодействие между сайтами (информация с 1 портала передаеться на другой)
Веб-сервис - веб ориентированная технология, которая позволяет программам общаться между собой используя стандартные форматы XML, JSON и посредством специального протокола SOAP и архитектурного стиля REST

------------------------- SOAP -------------------------
SOAP (Simple Object Access Protocol / простой протокол доступа к обьектам) - протокол обмена структурированными сообщениями в распределенной вычислительной среде
Протокол обмена сообщениями, с произвольным содержанием, но в формате XML
SOAP может использоваться с любым протоколом прикладного уровня (FTP, HTTP, HTTPS, OSI)
XSD (XML Schema Definition) - описывает структуру XML-документа и типы данных, которые там могут храниться
В веб-сервисе есть файл XSD с помощью которого можно задавать элементы, данные внутри XML и передавать их по сети интернет
WSDL (Web Services Description Language) - файл на языке WSDL и с таким же форматом, описывает сообщения, заголовки, события свойственные для конкретного веб-сервиса
Данный файл описывает структуру веб-сервиса
WSDL-файл - обязательный для SOAP-протокола и без него нельзя использовать этот протокол
XML (eXtensible Markup Language) - расширенный язык разметки
XML-файл хранит в себе информацию, которую можно передавать
Структурная единица XML - элементы, которые состоят из (<name attribute="best"> Content </name>)
открывающий тег, в треугольных скобках
аттрибуты
контент
закрывающий тег
Особенность тегов в XML:
1.) Только один корневой элемент
2.) Все должны содержать закрывающийся тег
3.) Названия тегов - регистрозависимые (<Message> Content </message> - wrong)
4.) Элементы не должны пересекаться (<b> Hello <i> all. </b> How are you ? </i> - wrong)
5.) Все значения атрибутов в кавычках (<person name="John" surname='Hi'/>
6.) Символы <, >, &, ', " - нельзя использовать в текствовых блоках
Вместо них нужно использовать их в текстовом формате: &lt - <, &gt - >, &amp - &, &apos - ', &quot - "
7.) Обьявление XML - всегда первая строка (<?xml version="1.0" encoding="UTF-8" standalone="yes"?>) (standalone - запрещены ли ссылки на внешние документы)

------------------------- REST -------------------------
REST (Representational State Transfer / передача представления о состоянии) - архитектурный стиль, который описывает структуру передачи данных у веб-серисов
К архитектурному стилю не применяються жесткие правила. На примере SOAP, тут есть WADL-файл, но он необязательный и можно работать без него
По сравнению с SOAP, REST позволяет записывать информацию в более удобном формате, который занимает меньше места и повышает производительность системы
RESTful - характеристика веб-сервисов, такие веб-сервисы, которые отвечают всем требованиям, которые предьявляет REST
В REST данные передаються в формате JSON (Javascript Object Notation)
Для JSON нет таких строгих правил как для XML, поэтому он более гибкий, а также занимает меньше памяти
Документы в формате JSON состоят из обьектов - неупорядоченное множество пар имя-значение
{"name": "John", "surname":"Joe"}
Имя - всегда строка, а значение - любой тип данных
Number и Boolean - можно не заключать в кавычки
{"index":123456789}
Если нужно передать несколько обьектов - они помещаются в массив
[{"name": "John"}, {"surname": "Doe"}, {"age": 30}]

Отличия REST и SOAP веб-сервисами
1.) REST поддерживает различные форматы (JSON, Text, XML, HTML), а SOAP - только XML
2.) REST работает только по HTTP и HTTPS протоколам, SOAP - поддерживает и другие
3.) REST может быть закеширован, SOAP - не может быть помещен в кеш
4.) REST - архитектурный стиль, у которого нет большого количества правил, которым он должен подчиняться, SOAP - протокол, ограниченный теми правилами, которые к нему применяются

REST - скорость, расширяемость, поддержка многих форматов и не такая привязка к правилам SOAP
SOAP - больше возможностей по безопасности (для тестировщиков - выше качество тестирования, благодаря WSDL, в которых описаны все запросы)
SOAP - более ресурсоемкий, медленнее, занимает больше места, разработка занимает больше времени
